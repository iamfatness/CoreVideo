CoreVideo

An OBS Studio plugin that captures live Zoom meeting video and audio, letting you stream or record any participant directly on your OBS canvas.

C++17 CMake 3.16+ OBS Studio Qt 6 Zoom Meeting SDK Windows macOS

Introduction

CoreVideo is an OBS Studio plugin that integrates the Zoom Meeting SDK to bring raw video and audio frames from a live Zoom meeting directly into OBS. No screen capture or virtual camera is involved — the plugin subscribes to raw I420 video frames and 48 kHz PCM audio from specific meeting participants or the active speaker, then converts and pushes those frames into OBS as a first-class source.

On Windows the Zoom SDK must run in its own process due to COM apartment and UI-thread constraints. A companion executable (ZoomObsEngine.exe) hosts the SDK and communicates back to the plugin via JSON over named pipes and shared memory for frame data. On macOS the SDK is loaded directly in-process.

Features

🎥

Raw Video Capture

I420 YUV frames from the Zoom SDK converted to BGRA for OBS textures.

🎙️

Raw Audio Capture

48 kHz PCM audio with configurable mono/stereo mixing modes.

👥

Participant Roster

Track all meeting participants with active-speaker detection.

🔄

Auto-Follow Speaker

Optionally follow the active speaker instead of a fixed participant.

🔒

JWT Authentication

Authenticates with the Zoom SDK using your SDK key, secret, and JWT token.

⚙️

Settings Dialog

Qt GUI dialog for credential management, accessible from OBS Tools menu.

🖥️

Multi-Platform

Windows (x64) and macOS (universal arm64 + x86_64) support.

🔗

IPC Engine (Windows)

Separate engine process communicates via named pipes for SDK isolation.

Architecture: Windows IPC Engine

On Windows, the Zoom SDK requires a dedicated UI-thread and COM STA context that conflicts with OBS's rendering thread. CoreVideo solves this by spawning ZoomObsEngine.exe — a separate process that owns the SDK — and communicating with it via two named pipes carrying newline-delimited JSON.

Color Space Conversion

The Zoom SDK delivers frames in I420 (YUV planar) format using BT.709 full-range colorspace. OBS consumes VIDEO_FORMAT_BGRA. The VideoDelegate performs this conversion on every received frame before calling obs_source_output_video().

IPC Protocol Reference

All messages are UTF-8 JSON terminated by \n, exchanged over two Windows named pipes. The plugin writes to \\.\pipe\ZoomObsPlugin_P2E (plugin-to-engine) and reads from \\.\pipe\ZoomObsPlugin_E2P (engine-to-plugin).

Plugin → Engine Commands

CommandPayloadDescription
init{"cmd":"init","jwt":"<token>"}Initialize Zoom SDK with JWT token
join{"cmd":"join","meeting_id":"<id>","passcode":"<pw>","display_name":"OBS"}Join a Zoom meeting
leave{"cmd":"leave"}Leave current meeting
subscribe{"cmd":"subscribe","source_uuid":"<uuid>","participant_id":<id>}Subscribe to a participant's video & audio
unsubscribe{"cmd":"unsubscribe","source_uuid":"<uuid>"}Stop frame delivery for a source
quit{"cmd":"quit"}Shut down the engine process

Engine → Plugin Events

EventPayloadDescription
ready{"cmd":"ready"}Engine started, pipes connected
auth_ok{"cmd":"auth_ok"}SDK authenticated successfully
auth_fail{"cmd":"auth_fail","code":<N>}Authentication failed with error code
joined{"cmd":"joined"}Successfully joined the meeting
left{"cmd":"left"}Meeting ended or left
frame{"cmd":"frame","uuid":"<uuid>","shm":"ZoomObsPlugin_<uuid>","w":W,"h":H}New video frame in shared memory
audio{"cmd":"audio","uuid":"<uuid>","shm":"ZoomObsPlugin_audio_<uuid>"}New audio chunk in shared memory
error{"cmd":"error","msg":"<reason>","code":<N>}SDK or meeting error

Shared Memory Layout

Frame data is written to named shared memory segments prefixed with ZoomObsPlugin_ followed by the source UUID. Video frames are raw BGRA pixels (width × height × 4 bytes); audio chunks are raw S16LE samples.

Installation

ℹ️ Pre-built releases coming soon
Pre-built binaries are not yet published. Install from source using the steps below.

Obtain the Zoom Meeting SDK

Download the Zoom Meeting SDK for your platform from the Zoom Developer Portal and place it at third_party/zoom-sdk/ so that third_party/zoom-sdk/h/zoom_sdk.h exists.

Configure CMake

cmake -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DZOOM_SDK_DIR=third_party/zoom-sdk \
  -DCMAKE_PREFIX_PATH="/path/to/obs-studio;/path/to/Qt6"

Build

cmake --build build --config Release

On Windows this also produces ZoomObsEngine.exe.

Install into OBS

cmake --install build --prefix "/path/to/obs-studio"

This copies the plugin module and the engine executable to obs-plugins/64bit/ and locale files to data/obs-plugins/obs-zoom-plugin/.

Launch OBS and configure credentials

Open OBS → Tools → Zoom Plugin Settings and enter your Zoom SDK Key, SDK Secret, and JWT token.

Configuration

Credentials (Settings Dialog)

Open Tools → Zoom Plugin Settings in OBS to save your Zoom developer credentials. These are persisted in the OBS global configuration file via obs_data_t.
