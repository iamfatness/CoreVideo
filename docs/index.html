CoreVideo

An OBS Studio plugin that captures live Zoom meeting video, audio, and screen share content, letting you stream or record any participant directly on your OBS canvas.

C++17 CMake 3.16+ OBS Studio Qt 6 Zoom Meeting SDK Windows macOS Linux

Introduction

CoreVideo is an OBS Studio plugin that integrates the Zoom Meeting SDK to bring raw video, audio, and screen share frames from a live Zoom meeting directly into OBS. No screen capture or virtual camera is involved — the plugin subscribes to raw I420 video frames and 48 kHz PCM audio from specific meeting participants or the active speaker, then converts and pushes those frames into OBS as first-class sources.

A companion executable ZoomObsEngine hosts the Zoom SDK in its own process to avoid threading and SDK isolation issues. This engine runs on all three supported platforms and communicates with the plugin via JSON over named pipes (Windows) or Unix domain sockets (macOS / Linux), with bulk frame data passed through shared memory.

A built-in TCP control server on port 19870 exposes a JSON API that lets external tools — scripts, dashboards, or broadcast automation systems — query participants, reassign sources, and control meeting state at runtime.

Architecture: IPC Engine

ZoomObsEngine runs as a separate process on all platforms, hosting the Zoom SDK to avoid threading constraints. The plugin communicates with it via newline-delimited JSON over a pair of IPC channels. Frame data is passed through named shared memory segments to avoid copying large buffers through the IPC channel.

Control Server API

The plugin starts a TCP server on 127.0.0.1:19870 accepting newline-delimited JSON. An optional token can be configured for authentication; if set, every request must include "token":"<value>".

ℹ️ Quick test
echo '{"cmd":"status"}' | nc 127.0.0.1 19870

IPC Protocol Reference

All messages are UTF-8 JSON terminated by \n. On Windows, two named pipes are used. On macOS and Linux, two Unix domain sockets are used instead. Frame data is always passed through named shared memory segments to avoid copying large buffers through the IPC channel.

Installation

ℹ️ Pre-built releases
Pre-built binaries are not yet published. Install from source using the steps below.

Obtain the Zoom Meeting SDK

Download from the Zoom Developer Portal and place it at third_party/zoom-sdk/ so that third_party/zoom-sdk/h/zoom_sdk.h exists. On Windows, CMake will auto-detect x64/arm64/x86 sub-layouts.

Configure CMake

cmake -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DZOOM_SDK_DIR=third_party/zoom-sdk \
  -DCMAKE_PREFIX_PATH="/path/to/obs-studio;/path/to/Qt6"

Build

cmake --build build --config Release

This builds obs-zoom-plugin and ZoomObsEngine on all platforms. On Windows, sdk.dll is also copied automatically if found.

Install into OBS

cmake --install build --prefix "/path/to/obs-studio"

Configure credentials and launch

Open OBS → Tools → Zoom Plugin Settings and enter your Zoom SDK Key, SDK Secret, and JWT token.
